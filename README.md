# Airport Customer Relations Management System - INSY6211 Assignment

# Project Overview
This repository contains the Systems Analysis and Design documentation for the Airport Customer Relations Management System (ACRMS). The project was completed as part of an academic assignment focusing on the application of structured analysis and design methodologies to a real-world airport system scenario.

# Assignment Context
A local airport has recently upgraded its infrastructure to accommodate international flights, anticipating an influx of tourists. To maintain high customer satisfaction, the airport requires a new Customer Relations Management System accessible via:

- Strategically placed airport kiosks
- Downloadable mobile application

# Core System Features
Information Services: Updates on flights, airport amenities, and available services
Navigation Capability: GPS navigation to various amenities and boarding gates
Emergency Services: Instant connection to medical and ambulatory services

# Learning Outcomes Addressed
- This assignment demonstrates proficiency in the following learning outcomes:
- Describing the purpose of system analysis and design in information systems development
- Describing the characteristics of iterative system development
- Describing the activities of Systems Analysis
- Identifying different kinds of stakeholders and their contributions to requirements definition
- Applying the event decomposition technique to identify use cases
- Interpreting and creating a domain model class diagram
- Developing use case descriptions

# Methodology
- The assignment applies structured Systems Analysis and Design principles including:
- Iterative Development Approach: Rationale for building the system through repeated cycles rather than a linear sequence
- Stakeholder Analysis: Identification of airport management, airline representatives, passengers, ground staff, and emergency personnel as key stakeholders
- Event Decomposition: Systematic identification of external events requiring system responses
- Domain Modeling: Conceptual modeling of real-world entities (Passenger and Flight) with defined attributes and relationships
